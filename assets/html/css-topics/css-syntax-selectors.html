CSS Syntax, Selectors & Specificity

CSS rules consist of selectors and declaration blocks. Selectors target HTML elements, while declaration blocks contain property-value pairs that define the styling.

Understanding CSS specificity is crucial for determining which styles will be applied when multiple rules target the same element.

Learning Process

  • Basic Selectors: Type, Class, and ID selectors
  • Combinators: Descendant, Child, and Adjacent sibling selectors
  • Specificity: How browsers determine which styles to apply

    • Basic Selectors:

    /* Type selector */
p {
  color: blue;
}

/* Class selector */
.text-red {
  color: red;
}

/* ID selector */
#main-title {
  font-size: 24px;
}

    Combinators:

    /* Descendant selector */
div p {
  color: green;
}

/* Child selector */
ul > li {
  list-style: none;
}

/* Adjacent sibling selector */
h2 + p {
  margin-top: 0;
}

    Specificity Example:

    /* Specificity: 0,0,1 */
p {
  color: blue;
}

/* Specificity: 0,1,0 */
.text-red {
  color: red;
}

/* Specificity: 1,0,0 */
#main-title {
  color: green;
}

    Real-World Example

    In a large website, you might have multiple CSS rules targeting the same elements. Understanding specificity helps you write more maintainable CSS and avoid unexpected styling issues.
